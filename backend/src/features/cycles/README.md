# Модуль циклов (`cycles`)

Модуль отвечает за автоматический и ручной сброс финансовых циклов пользователей. Основная логика находится в `cycle.js`, CLI-точка ручного запуска — в `manual-reset.js`, вспомогательные утилиты — в `utils.js`.

## Основная логика
1. При наличии активного цикла модуль проверяет, прошло ли с даты старта 30 суток.
2. Если цикл подошёл к завершению, предыдущий цикл закрывается, создаётся новый.
3. Остатки по сбрасываемым типам конвертов сохраняются в `remainders`.
4. После сброса пересчитываются лимиты типов и распределённые суммы нового цикла.

## Режимы запуска
- Автоматический режим:
  планировщик запускается раз в час через `cron.schedule('0 * * * *', ...)`.
- Ручной режим для тестирования:
  `npm run cycles:reset-now`

Ручная команда:
- вызывает ту же бизнес-логику, что и scheduler;
- работает в `force`-режиме и не ждёт 30 суток;
- проходит по всем пользователям;
- предназначена только для dev/test среды;
- запрещена в `production`.

## Полезные файлы
- `cycle.js` — общая логика scheduler-run и force-run.
- `manual-reset.js` — CLI для принудительного тестового сброса.
- `utils.js` — утилиты определения даты и правил сброса.
