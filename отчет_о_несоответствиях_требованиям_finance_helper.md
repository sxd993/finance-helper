### 2) Нет реализации перераспределения остатков цикла в накопления/инвестиции
- Требование: `4.8 Перераспределение остатков: в накопления, в инвестиции`.
- Факт:
  - Остатки цикла сохраняются в таблицу `remainders` (scheduler: `backend/src/features/cycles/cycle.js`).
  - В API остатков есть только чтение: `backend/src/routes/remainders/get-user-remainders.js`.
  - Endpoint пополнения `POST /api/converts/replenish-convert` работает с `convert_type_limits`, а не с сущностью `remainders`.
- Риск: обязательная операция из требований отсутствует (есть только просмотр, без перераспределения именно цикловых остатков).

План исправления:
1. Добавить backend-роут `POST /api/remainders/redistribute`.
2. В транзакции:
   - валидировать остаток по `remainder_id`/`cycle_id` и владельца,
   - списывать сумму из `remainders`,
   - зачислять в целевой тип (`saving`/`investment`) через обновление конверта(ов) и/или лимитов.
3. Добавить frontend-форму на странице остатков (выбор суммы и целевого направления).
4. После мутации инвалидировать запросы: `remainders`, `converts`, `convert-limits`, `overview`.
